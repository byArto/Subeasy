# SubEasy → Google Play — материалы для публикации

Готовый набор для копирования в Play Console. Источник правды по данным — страница `/privacy`.
Аккаунт: **личный** → обязателен закрытый тест **12 тестеров × 14 дней** до продакшна.
Домен (канонический): **www.subeasy.org** — голый `subeasy.org` редиректит (307) на www, поэтому везде (PWABuilder, assetlinks, Privacy URL) используем именно `www`.
Package name (Android applicationId): **`org.subeasy.twa`** (тот же вписать в PWABuilder и в assetlinks.json).

## 2. Форма Data Safety (Безопасность данных)

> Google наказывает за занижение. Заполнять честно по `/privacy`. ВАЖНО: Telegram-аналитика и fingerprinting **не запускаются** в Android-приложении (только внутри Telegram) — в TWA из аналитики работает только Vercel Analytics (обезличенная).

**Собирает ли приложение данные пользователя?** — ДА.

| Тип данных | Собирается | Передаётся 3-м лицам | Обязательно | Зачем |
|---|---|---|---|---|
| Email | Да | Нет | Нет (только при входе) | Аутентификация, синхронизация |
| Имя / username (Telegram) | Да, если подключён Telegram | Нет | Нет | Профиль, уведомления |
| «Другая финансовая информация» (суммы подписок, которые вводит пользователь) | Да | Нет | Да (суть приложения) | Функции приложения |
| Действия в приложении (App interactions) | Да | Нет | Нет | Аналитика, улучшение продукта |
| Идентификаторы устройства / прочее | Да (Vercel Analytics, обезличенно) | Нет | Нет | Аналитика |

**«Передаётся» (shared)** — ВЕЗДЕ НЕТ: Supabase и Vercel выступают как обработчики (наши сервис-провайдеры), а не как третьи лица для своих целей.

**Практики безопасности:**
- Данные шифруются при передаче (HTTPS) — **ДА**.
- Пользователь может запросить удаление данных — **ДА** (в приложении есть удаление аккаунта → `/api/account/delete`).
- Соответствие Families Policy — **НЕТ** (приложение не для детей).

**Privacy policy URL:** `https://www.subeasy.org/privacy`

## 4. Графика — что готово и что доделать

| Ассет | Требование Google | Статус |
|---|---|---|
| Иконка приложения | 512×512, 32-bit PNG с альфа, ≤1 МБ | ✅ `SubEasy_icons_final (1)/icon-512x512.png` |
| Feature graphic | **1024×500**, JPG или 24-bit PNG (без альфы) | ❌ сделать (основа: `logo final.png` 1024×1024) |
| Скриншоты телефона (2–8 шт.) | 320–3840 px; длинная сторона ≤ 2× короткой; JPG или 24-bit PNG без альфы | ⚠️ `subeasy screen/*.PNG` = 1320×2868 → **слишком высокие** (макс 1320×2640), подрезать/в рамку; `IMG_4895/4935.jpg` по пропорциям ОК |

Рекомендация по скриншотам: 4–6 чистых кадров главных экранов (дашборд с суммой, список подписок, добавление, календарь/аналитика). Привести к ≤2:1 и без прозрачности.

## 5. Технические предпусковые проверки

- [ ] В Vercel: домен www.subeasy.org боевой; `NEXT_PUBLIC_APP_URL=https://www.subeasy.org` (сейчас в env пусто).
- [ ] После сборки PWABuilder вписать реальный **SHA-256** в `public/.well-known/assetlinks.json` (сейчас заглушка) и передеплоить.
- [ ] Проверить, что `https://www.subeasy.org/.well-known/assetlinks.json` открывается и отдаёт JSON.
- [ ] Lighthouse PWA ≥ 80 (PWABuilder проверит сам).
- [ ] (P1) Кнопка «Назад» для Android — закрывать открытое окно, а не приложение (тест на реальном устройстве).

## 6. Порядок действий

1. Доделать графику (раздел 4) + проверки (раздел 5, кроме fingerprint).
2. Задеплоить сайт с обновлённым манифестом и assetlinks.
3. PWABuilder: ввести https://www.subeasy.org → Package for Stores → Android → скачать `.aab` + signing key.
4. Вписать SHA-256 в assetlinks.json → передеплой.
5. Play Console: создать приложение, заполнить карточку (раздел 1), Data Safety (раздел 2), App content (раздел 3), залить графику и `.aab`.
6. Closed testing: добавить 12 тестеров, держать 14 дней непрерывно.
7. Через 14 дней → Apply for production access → ответить на вопросы → продакшн-ревью.
