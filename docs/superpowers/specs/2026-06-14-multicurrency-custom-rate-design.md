# Мультивалютность (CIS-набор) + возврат «Свой курс» — Design

**Дата:** 2026-06-14
**Статус:** утверждён к реализации (ожидает финальной вычитки пользователем)

## Цель

1. Вернуть в настройки функцию **«Свой курс»** (ручной обменный курс), удалённую в v1.7.11 (commit `0bd77b9`).
2. Добавить новые валюты так, чтобы конвертация работала **корректно и масштабируемо**: BYN, KZT, UAH, AMD, KGS, UZS, GEL — поверх существующих RUB / USD / EUR.

## Контекст / текущее состояние

- Конвертация прибита к трём валютам через **два скалярных курса**:
  `convertCurrency(amount, from, to, usdRate, eurRate)` ([src/lib/utils.ts:26](../../../src/lib/utils.ts)).
  Эта сигнатура используется в ~15 местах (итоги на главной, календарь, аналитика, шеринг, экспорт, отчёты, cron, Telegram-отчёт).
- `/api/rate` ([src/app/api/rate/route.ts](../../../src/app/api/rate/route.ts)) тянет с ЦБ только USD и EUR.
- Типы `Currency`/`DisplayCurrency = 'RUB' | 'USD' | 'EUR'` ([src/lib/types.ts:1-2](../../../src/lib/types.ts)).
- Пикер валют `CURRENCY_OPTIONS` сейчас содержит только **RUB + USD** (EUR полуудалён: ветка `locked === 'EUR'` — мёртвый код).
- Серверный cron ([src/app/api/cron/notify/route.ts](../../../src/app/api/cron/notify/route.ts)) и Telegram-отчёт читают из БД только `exchange_rate` + `eur_exchange_rate` и имеют собственный RUB/USD/EUR-конвертер.
- Настройки синкаются в Supabase `user_settings` только двумя скалярными колонками курса.
- **PRO упразднён полностью** — никаких PRO-бейджей/блокировок в выборе валюты быть не должно.
- i18n-ключи для «Свой курс» уже существуют: `settings.currency.customRate`, `customRateHint`, `usdRub`, `manualRate`, `cbrf` ([src/lib/translations.ts:482-486](../../../src/lib/translations.ts)).
- Автотестов на конвертацию нет.

## Архитектурное решение: карта курсов

Заменяем модель «два скалярных курса» на **карту «рублей за единицу»**: `Record<Currency, number>`, где `RUB = 1`.

Конвертация любой пары становится тривиальной и всегда корректной:

```
converted = amount × rates[from] / rates[to]
```

Это устраняет необходимость плодить `eurRate`-подобные параметры и делает добавление валют безопасным.

### Корректность Nominal (ключевой момент)

ЦБ (`cbr-xml-daily.ru/daily_json.js`) отдаёт по каждой валюте `Value` и `Nominal`. У части CIS-валют `Nominal ≠ 1` (например, KZT — «рублей за 100 тенге»). Поэтому «рублей за 1 единицу» = `Value / Nominal`, и **Nominal читается динамически из фида**, не хардкодится. Текущий код это игнорирует (USD/EUR оба Nominal 1), иначе новые валюты считались бы в 100×/10000× неверно.

## Список валют (10)

| Код | Символ | Название | Прим. Nominal в ЦБ |
|-----|--------|----------|--------------------|
| RUB | ₽ | Рубль (база) | — (rate = 1) |
| USD | $ | Доллар США | 1 |
| EUR | € | Евро | 1 |
| BYN | Br | Белорусский рубль | 1 |
| KZT | ₸ | Казахстанский тенге | 100 |
| UAH | ₴ | Украинская гривна | 100 |
| AMD | ֏ | Армянский драм | 100 |
| KGS | сом | Киргизский сом | 100 |
| UZS | сўм | Узбекский сум | 10000 |
| GEL | ₾ | Грузинский лари | 1 |

Nominal в таблице — справочно; код берёт его из фида. Валюта, отсутствующая в фиде в конкретный день, не попадает в карту и берёт фоллбэк (кеш/дефолт).

## Миграция данных
- **localStorage** (`neonsub-settings`): при чтении, если `manualRates` отсутствует, но `useManualRate = true` и есть `exchangeRate`/`eurExchangeRate` — засеять `manualRates.USD`/`manualRates.EUR`. Кэш курса мигрируется со старого формата.
- **БД**: при отсутствии `rates` сервер использует старые колонки (USD/EUR/RUB), как сейчас.

## Вне объёма (YAGNI)
- Не трогаем RLS и ротацию ключей (отдельная задача).
- Без исторических курсов/графиков — только текущая конвертация.
- Без выбора источника курса (только ЦБ + ручной).
- Без PRO-логики любого рода.

## Риски
- Самое широкое изменение — миграция `convertCurrency` на карту в ~15 файлах (механически, но затрагивает много мест) и редизайн пикера.
- DB-миграция `rates jsonb` — отдельный шаг, который нужно применить к Supabase.
- Серверная и клиентская конвертация должны использовать одну и ту же карту, иначе суммы в пушах разойдутся с UI — закрывается синком `rates`.

## Критерии готовности
- Все 10 валют выбираются как валюта отображения; суммы конвертируются корректно везде (главная, карточки, деталь, календарь, аналитика, бюджет, шеринг, экспорт, отчёты).
- KZT/UAH/AMD/KGS/UZS считаются с учётом Nominal (нет 100×-ошибок).
- «Свой курс» включается/выключается, ручные курсы сохраняются и применяются; авто-режим возвращает курс ЦБ.
- Пуш-напоминания и Telegram-отчёт показывают суммы, совпадающие с UI, для новых валют.
- `npm run build` проходит без ошибок.
- Старые данные пользователей (USD/EUR-скаляры) не ломаются.
